# Playwright Teaching Plan

## Purpose

This repo should support practical teaching from the first session. Students should not only read concepts. They should clone the repository, run the local app, execute tests, and practice one small automation unit at a time.

## Training Model

Each section should be taught as a practical unit:

1. introduce one small concept
2. show one working coded example
3. let students run the same example
4. give one closely related exercise
5. move forward only after that unit is clear

## Repo Design

The repository should have two parallel tracks:

- `docs/` for reference material and concept explanation
- `units/` for hands-on examples and student exercises

## Practical Learning Order

1. Getting started with the local app and first Playwright test
2. Locators and basic interactions
3. Assertions and validation
4. Small UI test flows
5. Framework basics with page objects
6. API testing with Playwright
7. Config, test data, logging, and reporting
8. Build a proper Playwright framework with page objects, fixtures, and reusable structure

## What Each Unit Should Include

- a short unit `README.md`
- one instructor example test
- one student exercise starter file
- only enough code to teach that unit clearly

## Practical Outcome

By following this structure, students can learn by doing instead of waiting until later sessions to start writing automation code.
